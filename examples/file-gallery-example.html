Галерея файлов gtsAPIFile - Примеры использования

Внимание: Для работы примеров необходимо настроить серверную часть API и подключить Vue.js приложение.

3. Программная работа с API
Пример использования FileGalleryAPI для программной работы с файлами.
JavaScript код: 
import { FileGalleryAPI, fileUtils } from 'pvtables'

// Создание экземпляра API
const api = new FileGalleryAPI('/api/file-gallery')

// Загрузка списка файлов
async function loadFiles() {
  try {
    const response = await api.getFiles({
      parent: 123,
      class: 'modResource',
      list: 'gallery',
      limit: 20
    })
    
    if (response.success) {
      console.log('Файлы:', response.data.files)
      return response.data.files
    }
  } catch (error) {
    console.error('Ошибка:', error)
  }
}

// Загрузка файла
async function uploadFile(file) {
  // Валидация файла
  const validation = fileUtils.validateFile(file)
  if (!validation.valid) {
    alert(validation.error)
    return
  }
  
  try {
    const response = await api.uploadFiles([file], {
      parent: 123,
      class: 'modResource',
      list: 'gallery'
    })
    
    if (response.success) {
      console.log('Файл загружен:', response.data.files[0])
    }
  } catch (error) {
    console.error('Ошибка загрузки:', error)
  }
}
4. Обработка событий галереи
Пример обработки различных событий компонента FileGallery. 
<template>
  <FileGallery
    :parent-id="resourceId"
    parent-class="modResource"
    @file-selected="onFileSelected"
    @files-uploaded="onFilesUploaded"
    @file-updated="onFileUpdated"
    @file-deleted="onFileDeleted"
    @error="onError"
  />
</template>

<script>
export default {
  data() {
    return {
      resourceId: 123,
      selectedFile: null
    }
  },
  
  methods: {
    onFileSelected(file) {
      this.selectedFile = file
      console.log('Выбран файл:', file.name)
    },
    
    onFilesUploaded(files) {
      console.log(`Загружено ${files.length} файлов`)
      // Обновить список или показать уведомление
    },
    
    onFileUpdated(file) {
      console.log('Файл обновлен:', file.name)
    },
    
    onFileDeleted(file) {
      console.log('Файл удален:', file.name)
      if (this.selectedFile?.id === file.id) {
        this.selectedFile = null
      }
    },
    
    onError(message) {
      console.error('Ошибка галереи:', message)
      // Показать уведомление об ошибке
    }
  }
}
</script>

6. Настройка серверной части
Пример настройки API маршрутов и контроллера на сервере.
PHP код для настройки API: 
<?php
// Создание правила API
$rule = $modx->newObject('gtsAPIRule');
$rule->fromArray([
    'point' => 'file-gallery',
    'description' => 'API для галереи файлов',
    'controller_class' => 'gtsAPIFileGalleryController',
    'controller_path' => MODX_CORE_PATH . 'components/gtsapi/controllers/',
    'active' => 1,
    'authenticated' => 1,
    'groups' => 'Administrator,Editor',
    'permitions' => 'file_manager'
]);
$rule->save();

// Добавление действий
$actions = [
    [
        'gtsaction' => 'list',
        'authenticated' => 1,
        'active' => 1
    ],
    [
        'gtsaction' => 'upload', 
        'authenticated' => 1,
        'permitions' => 'file_upload',
        'active' => 1
    ],
    [
        'gtsaction' => 'update',
        'authenticated' => 1, 
        'permitions' => 'file_edit',
        'active' => 1
    ],
    [
        'gtsaction' => 'delete',
        'authenticated' => 1,
        'permitions' => 'file_delete', 
        'active' => 1
    ]
];

foreach ($actions as $actionData) {
    $action = $modx->newObject('gtsAPIAction');
    $action->fromArray(array_merge($actionData, [
        'rule_id' => $rule->get('id')
    ]));
    $action->save();
}
Системные требования
Для работы галереи файлов необходимо:
  • MODX Revolution 2.8+
  • Компонент gtsAPI
  • Vue.js 3.x
  • PVTables библиотека
  • PHP 7.4+ с расширениями GD или ImageMagick
  • Настроенные медиа-источники MODX
Безопасность:
  • Настройте права доступа через систему разрешений MODX
  • Ограничьте типы загружаемых файлов
  • Установите лимиты на размер файлов
  • Используйте отдельный домен для файлов пользователей
