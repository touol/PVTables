# Отладка проблемы с изменением размера колонок

## Проблема
При изменении размера колонок мышкой в dev режиме стили не применяются к колонкам, растягивается вся таблица.

## Возможные причины

### 1. Проблема с nth-child селектором
В методе `resizeTableCells` используется селектор `nth-child(${index + 1})`, который может неправильно работать если есть дополнительные колонки (checkbox, tree expander).

### 2. Проблема с виртуальным скроллингом
Если включен виртуальный скроллинг, селектор может быть неправильным.

### 3. CSP в dev режиме
Content Security Policy может блокировать динамические стили.

## Решение
Нужно проверить:
1. Правильность индексации колонок
2. Селектор для виртуального скроллинга
3. Применение стилей через styleElement

## Тестирование
1. Открыть dev tools
2. Попробовать изменить размер колонки
3. Проверить в Elements, создается ли `<style>` элемент в `<head>`
4. Проверить содержимое этого элемента
5. Проверить, применяются ли стили к `<th>` и `<td>`
